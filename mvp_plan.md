# MVP Стратегия: Этапы разработки

Принципы построения MVP:

- Каждый этап приносит ценность - пользователь может использовать на любом этапе
- Инкрементальные изменения - не революционные, а эволюционные
- фокус на ядро - места хранения + предметы + поиск + QR-коды
- Простота первого - минимум полей и функций
- Расширение по обратной связи - добавляем на основе использования

## Фаза 1: MVP Минимум (6 недель) - "Базовый учет"

Цель: Создать простейшую систему учета предметов по местам хранения

Функционал:

Создание "мест хранения" (Location)

1. Название (например: "Балкон", "Коробка #1", "Полка 2")
2. Описание
3. Фото места

Добавление "предметов" в места

- Название предмета
- Описание/примечание
- Фото предмета
- Привязка к месту хранения
- Дата добавления

Базовый поиск

- Поиск по названию предмета
- Поиск по названию места

Просмотр содержимого места

- Список всех предметов в месте с фото
- Быстрое редактирование

Технология:

- Мобильное приложение (Android + iOS)
- Локальное хранилище (SQLite) - полный офлайн режим

Ценность для пользователя:

    ✅ Можно начать организовывать балкон прямо сейчас

    ✅ Хотя бы текстовые подсказки где что лежит

    ✅ Фото помогут вспомнить содержимое

Метрики успеха:

- User adds 20+ items
- Daily active session time > 5 min
- Чувство облегчения при поиске вещей

## Фаза 2: QR-коды (8 недель) - "Физическая интеграция"

Цель: Превратить места хранения в QR-метки, сканируемые камерой

Новый функционал:

Генерация уникального QR-кода для каждого места

- Встроенный QR-генератор
- Предустановленные размеры печати
- Сканирование QR-кода
- Открытие места и его содержимого одним скан-кодом
- Камера как первичный способ навигации

Быстрое добавление предмета через сканирование

- Отсканировал QR места → сразу добавить предмет

Печать этикеток

- На непосредственно на полку, коробку
- С малым размером (стикер 5x5 см)

Технология:

- Интеграция QR библиотеки (QR.js, zxing)
- Встроенный генератор (QRCode.js)
- Локальный Print API браузера или native приложения

Ценность для пользователя:

    ✅ Клейкая этикетка на коробку - визуальный индикатор

    ✅ Один скан = мгновенный доступ к содержимому

    ✅ Более "технологичное" решение, чем текстовые списки

Метрики успеха:

    QR коды распечатаны и наклеены на 50%+ мест

    Сканирование - основной способ открытия содержимого

    User adoption QR workflow > 60%
